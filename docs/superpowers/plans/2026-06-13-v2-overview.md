## Workstreams

### W1 — WPF visual refresh (overlay; sequential, in waves)
1. **Foundation** *(in progress)*: plumb unused per-wheel fields (TireTemp/SlipRatio/SlipAngle/CombinedSlip)
   onto `TelemetryReadout` + VM (tests); add SkiaSharp package; add color-preset + custom-accent theming
   (config + `ThemePalette` presets + settings UI). Widgets adopt the palette during redesign.
2. **Refreshed core widgets**: G-force (color-on-exit + tether/trail/peak), chart area-fill + animation,
   apply Theme A colors + B proportions across existing widgets.
3. **New exploration widgets**: gear-drum (03), boost turbine (04) + induction-flow (12), steering horizon
   (08), speed tape (09), fuel arc (10, hidden default), shift ladder (11, customizable), speed slipstream
   bg (02). Each is a draggable/customizable `WidgetId` like the rest; some replace/alternate existing ones.
4. **Tire widget**: per-corner temp color tiles + grip headroom (uses the newly plumbed fields).

### W2 — QoL
- **Setup/diagnostics helper**: in-app check (UDP bind ok? packets/sec? hints on UWP loopback/port) — surfaced in settings/status.
- **Gear-ratio readout**: estimated per-gear ratio (WheelRotationSpeed vs Rpm) widget.
- **CSV export = CLI decode tool**: a `fh6 decode <capture.jsonl> [--out csv]` subcommand that decodes the
  raw UDP capture to a CSV of parsed fields (exposes the parser as a CLI tool).

### W3 — Node map-replay web app (independent sub-project)
A lightweight Node web app that renders the FH6 world map (the seasonal AVIFs — browsers render AVIF natively)
and **replays a capture from top-down** with telemetry: car path/marker over the map, play/scrub timeline,
telemetry readouts, pan/zoom, season switch. Reuses the JSONL capture format; decodes packets in JS (or
consumes the CSV from the W2 decode tool). Calibration: auto-fit the path bounds + manual scale/offset nudge
(web makes pan/zoom/nudge easy).

## Sequencing
W3 runs in parallel (separate folder/worktree). W1 is the heavy sequential core (foundation → waves). W2 is
small and slots between W1 waves. Each WPF change is verified by running the overlay + screenshot (CLAUDE.md rule).
